# Dataplane Interface

## Overview

The Dataplane Interface provides a web-based application used to operate and manage the AI Fabrix Dataplane service. It allows users to configure integrations with external systems, manage integration templates, monitor system activity, and review operational records.

The interface is organized into several primary sections that expose the operational capabilities of the Dataplane environment.

---

## Dashboard

### Description

The Dashboard provides an overview of the Dataplane environment and operational activity.

It presents system metrics, environment status information, and summaries of ongoing system activity.

### What Users Can Do

Users can:

- View overall environment information
- Monitor system health indicators
- Review synchronization activity
- Observe system metrics
- Access quick operational actions related to system configuration or integration activity

### Typical Workflows

Typical uses of the dashboard include:

- Checking the operational status of the Dataplane
- Reviewing recent system activity
- Monitoring synchronization operations
- Navigating to specific systems or operational areas for further investigation

### Governance Considerations

Information displayed in the dashboard may include operational data related to synchronization processes, system activity, and integration operations.

---

## External Systems

### Description

The External Systems section manages connections between the Dataplane and external platforms or data sources.

This area provides visibility into configured systems and allows users to review or manage their associated data sources.

### What Users Can Do

Users can:

- View the list of configured external systems
- Access details for a specific system
- Review associated data sources connected to each system
- Manage integration connections to external platforms

### Typical Workflows

Typical workflows include:

- Adding new external systems
- Reviewing configured integrations
- Accessing system-specific configuration details
- Managing connected data sources

### Governance Considerations

External systems represent integration points with external platforms. Configuration of these systems may involve credentials, connection parameters, or access configurations that affect how data is accessed and synchronized.

---

## Integration Templates

### Description

Integration Templates provide reusable configurations used to define integration behavior.

Templates allow users to define standardized integration structures that can be applied across multiple systems.

### What Users Can Do

Users can:

- View available integration templates
- Access template configuration details
- Manage template definitions used by integrations

### Typical Workflows

Typical workflows include:

- Reviewing available integration templates
- Navigating to template configuration pages
- Managing templates used in integration workflows

### Governance Considerations

Templates influence how integrations operate. Modifications to templates may affect integration behavior across systems that use those templates.

The Dataplane Interface organizes functionality into the following primary sections:
